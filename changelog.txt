# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.4] - 2024-12-19

### Added
- **WooCommerce Subscriptions Support**: Added dedicated product handler for simple subscription products
- **Sold Individually Support**: Complete implementation of WooCommerce "Sold individually" product setting
- **Variable Product Variations**: Enhanced support for "Sold individually" setting on both parent and individual variation levels
- **Quantity Limiting**: Automatic quantity field limiting to 1 for products marked as "Sold individually"
- **Visual Feedback**: Clear indicators and tooltips for sold individually products
- **Validation System**: Comprehensive validation rules for sold individually products

### Enhanced
- **Product Handlers**: Extended all product handlers (simple, variable, subscription) to include sold individually information
- **Frontend UI**: Updated React components to respect sold individually settings with proper visual feedback
- **Validation System**: Added centralized validation rules for sold individually products
- **Variable Products**: Fixed support for both parent-level and variation-level sold individually settings
- **User Experience**: Improved quantity field behavior and user feedback for sold individually products

### Fixed
- **Variable Product Variations**: Fixed issue where variable product variations weren't respecting sold individually settings
- **Parent vs Variation Logic**: Properly implemented OR logic for parent and variation sold individually settings
- **Frontend Data Flow**: Ensured sold individually information is properly passed from backend to frontend
- **Validation Messages**: Added appropriate validation messages for different product types

### Technical Improvements
- **Code Standards**: All new code follows WordPress Coding Standards and Plugin Handbook guidelines
- **Documentation**: Enhanced code documentation with proper PHPDoc comments
- **Internationalization**: Added proper translation strings for all new features
- **CSS Styling**: Added dedicated styles for sold individually product indicators
- **Error Handling**: Improved error handling and user feedback throughout the system
